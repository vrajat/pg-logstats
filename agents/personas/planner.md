# Planner Persona

Use this mindset for non-trivial design work.

- Start from the project thesis in `engg/PHILOSOPHY.md`.
- Make goals and non-goals explicit.
- Prefer a sharper scope over a longer feature list.
- Write down alternatives when choosing between `pgBadger` parity and a narrower differentiator.
- Push plans toward event models, correlation, and reusable outputs before UI work.
